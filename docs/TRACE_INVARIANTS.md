# Trace Invariants — ENFORCED 🔒

## Core Invariant

> Every mission execution MUST carry a `trace_id` from creation to final output.

## trace_id Format

- Pattern: `tr-{12_hex_chars}`
- Example: `tr-a4504aa8d53b`
- Length: exactly 15 characters
- Generated by: `core/observability/event_envelope.generate_trace_id()`

## Propagation Chain

```
mission_system.submit()  → generates trace_id
  ├─ decision_trace.trace_id = trace_id
  ├─ set_trace(trace_id, mission_id)  [thread-local]
  │
  ├─ tool_executor  → emit_quick("tool", "tool_call", ...) [reads from thread-local]
  ├─ CanonicalAction.trace_id = trace_id
  │
  └─ result_aggregator  → FinalOutput.trace_id = trace_id
      └─ API response.result_envelope.trace_id = trace_id
```

## Where trace_id Must Exist

| Component | Field | Required |
|-----------|-------|----------|
| EventEnvelope | trace_id | YES — constructor param |
| CanonicalAction | trace_id | YES — constructor param |
| decision_trace | trace_id | YES — set at submit |
| FinalOutput | trace_id | YES — from decision_trace |
| API result_envelope | trace_id | YES — from FinalOutput |
| Thread-local context | get_trace_id() | During execution |

## Failure Mode

If trace_id is missing:
- EventCollector.emit_quick() silently no-ops (no trace context = no event)
- FinalOutput.trace_id defaults to "" (visible in API response)
- Tests FAIL on missing propagation

This is NOT silent failure — the API response exposes missing trace_id.
