# SaaS MVP Scope — Reasoning Structure

## Step 1: Core Value Identification
- What is the ONE thing this product must do well?
- Remove everything that doesn't serve this core

## Step 2: Feature Prioritization (MoSCoW)
- MUST: Without these, the product cannot be sold
- SHOULD: Important but can wait for v1.1
- COULD: Nice to have, do not build yet
- WON'T: Explicitly excluded — document why

## Step 3: Scope Reduction
- For each MUST feature, find the simplest implementation
- Challenge every feature: "Can we launch without this?"
- Target: 3-5 must-have features maximum

## Step 4: Dependency Mapping
- What APIs, services, or tools are required?
- Any vendor lock-in risks?
- Open-source alternatives?

## Step 5: Risk Assessment
- Technical risks (complexity, unknowns)
- Market risks (demand, timing, competition)
- Resource risks (skills, time, budget)

## Step 6: Development Estimate
- Break into components with hour estimates
- Flag components requiring specialized skills
